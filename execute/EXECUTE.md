# EXECUTE — Stage 4 of the Flywheel
## Operator Engagement Researcher · Execution Layer

EXECUTE is where intelligence becomes action. The Researcher built the picture. PRAECEPTOR built the plan. EXECUTE is where the operator runs the engagement — and where the plan meets reality.

This stage does not route. It equips. The four functions here give the operator the frameworks to run the execution layer of any engagement without reverting to instinct alone.

---

## What EXECUTE Provides

Four functions — each adapted from a full operating discipline — deployed specifically for the engagement execution context:

| Function | What It Covers |
|---------|---------------|
| **Delivery** | Running the engagement with urgency, discipline, and visibility |
| **Finance** | Reading the client's financials while running the engagement |
| **Partnerships** | Building trust with the client's team during execution |
| **Marketing** | Positioning recommendations and navigating organizational resistance |

Each function has an identity (who it is), rules (what it enforces), examples (what it looks like done well and done badly), and reference files (the frameworks it draws on).

---

## How EXECUTE Connects to the Flywheel

```
PRAECEPTOR (develop)
    ↓ engagement/plan.md — 30/60/90 plan + handoff envelope
EXECUTE (operator runs the engagement)
    ↓ Execution reveals friction → mentor-brief/brief.md → PRAECEPTOR re-engaged
    ↓ Execution reveals intelligence gaps → back to THE RESEARCHER
    ↓ Engagement ends → CALIBRATION SESSION
```

**EXECUTE does not close the loop — calibration does.** When the engagement ends, run a calibration session. The calibration updates memory, records what the system got wrong, and makes the next engagement in this domain start smarter.

---

## When to Route Back

EXECUTE is not a terminal stage. Route back when:

| Condition | Route to |
|-----------|---------|
| Capability friction in the engagement | PRAECEPTOR (re-engage) |
| New intelligence gap surfaces mid-engagement | THE RESEARCHER |
| New domain signal becomes relevant | SOURCE |
| Engagement complete | CALIBRATION |

The engagement log lives in `mentor-brief/brief.md`. Every friction block that surfaces during execution is written there automatically. PRAECEPTOR reads it on re-engagement.

## Workflow Reference

Each EXECUTE function operates against a canonical workflow that defines the full protocol for that discipline. These workflows are the operating manuals the functions run against — not the engagement plan, which is specific to this client, but the underlying operating discipline that the function brings to every engagement.

| Function | Canonical Workflow | What the workflow provides |
|---------|-------------------|---------------------------|
| **Delivery** | `workflows/WF_02_ENGAGEMENT_DELIVERY.md` | Diagnostic phases, system installation protocol, reporting cadence, mentor brief loop, Amp It Up communication standard |
| **Finance** | `workflows/WF_06_FINANCIAL_ARCHITECTURE.md` | Financial Intelligence principles, SDE mechanics, ROIC framework, DSCR, pre-mortem on acquisition decisions |
| **Partnerships** | `workflows/WF_01_BUSINESS_DEVELOPMENT.md` | Trust equation, SPIN discovery, STATE framework, relationship architecture during execution |
| **Marketing** | `workflows/WF_02_ENGAGEMENT_DELIVERY.md` (Stage 4) | Findings delivery, change navigation, Rider/Elephant framework, early win design |

**When to load:** On first invocation of a function in a new engagement, load the workflow's relevant section alongside the dept files. The workflow tells the function *what* its operating standards are. The engagement plan tells it *what to do* for this specific client.

**Loading instruction:** When CLAUDE.md routes to a dept at Stage 4, load `execute/[dept]/` files first, then load the dept's canonical workflow and read the section titled "Department Responsibilities" or the engagement-execution-specific stages.

---

## What EXECUTE Is Not

- It is not a management consulting methodology. It is not prescriptive about how to run any specific engagement.
- It is not a replacement for PRAECEPTOR. PRAECEPTOR plans. EXECUTE equips the operator to run the plan.
- It is not static. When execution reveals that the plan needs adjustment, route to PRAECEPTOR. When execution reveals that research was incomplete, route to THE RESEARCHER.

---

*The plan was built with what was known before the engagement began. EXECUTE is what keeps the operator grounded when the engagement reveals what wasn't.*
